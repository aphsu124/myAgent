### 5. 報告品質與自癒規範 (Engineering Standards) - 2026-04-01 更新
為解決報告內容截斷與 AI 偷懶問題，系統已全面導入「語義優先」自癒模式：
- **內容完整性保護**：
    - 內容若少於 500 字或 HTML 檔案小於 2500 bytes，系統**嚴禁覆蓋**主網頁 (index.html)。
    - 物理截斷（如移除簽名）僅限於字串最後 200 字元內，避免誤殺正文。
- **語義優先審查 (Validator V3)**：
    - 審查重心從字數轉向「深度特徵」：偵測文中是否包含「影響、風險、建議、策略、動態」等管理分析詞彙。
    - **事實對標**：不再強求關鍵字精準匹配，改為檢查報告與搜尋來源的語義關聯度。
- **高容錯解析機制**：
    - 使用全域掃描 regex `DATA_JSON[:：]?\s*({.*?})` 提取數據，不限位置，且需能處理被截斷的 JSON 代碼塊。
    - 物理截斷：優先尋找最後一個句號 `。` 作為終點，確保 PDF 不會出現「話說一半」的現象。
- **發布順序規範**：
    - 必須遵循「產出 -> 同步 -> 校對檔案大小 -> 發送 LINE」的順序，確認網頁 100% 更新後才發通知。
- **網頁即時性保障** (2026-04-02 新增)：
    - 產出的 HTML 必須包含 `<head>` 防快取標籤 (`no-cache`)，防止瀏覽器顯示舊版資訊。
- **指令化執行支援** (2026-04-02 新增)：
- 手動檢查規範 (2026-04-02 新增)：
    - 凡老闆指令要求「執行程式檢查」，Jarvis 必須啟動「三方 AI 議會會診」(Gemini + Claude + GPT-4o)，分別從主控、邏輯、除錯三個維度產出聯合檢查報告。    - `daily_palm_report.py` 支援 `--morning` 與 `--full` 參數，允許手動強制產出特定時段報告。

### 6. 新任務商量規範 (New Task Protocol) - 2026-04-01 新增
為確保所有新功能具備高度穩定性，任何新種類任務在【需求澄清階段】必須與老闆討論以下除錯細項：
- **定義合格標準**：明確列出任務成功的實體指標（檔案大小、關鍵字、結構）。
- **預判失敗模式**：討論 AI 或腳本可能出錯的點（如內容截斷、數據誤判）。
- **訂定自癒邏輯**：確認 Guardian 偵測到錯誤後的具體行動（重寫、報警、或保留舊檔）。
